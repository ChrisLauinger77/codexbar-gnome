## Español

¡Gracias por querer mejorar CodexBar! Para mantener el proyecto organizado y seguro, por favor sigue estas pautas:

### 1. PRs generadas por IA
Aceptamos Pull Requests (PRs) que utilicen IA para su creación, pero con condiciones:
- **Verificación obligatoria:** Debes haber testeado los cambios localmente y asegurarte de que funcionan.
- **Lectura crítica:** Debes haber leído y entendido el código generado. No aceptaremos código "caja negra".
- **Cambios quirúrgicos:** Evita reescribir archivos enteros o cambiar indentaciones y formatos innecesariamente. Si solo quieres añadir una función, cambia solo esa parte.

### 2. Ediciones del Mantenedor
Al abrir una PR, **asegúrate de marcar la casilla "Allow edits from maintainers"** (Permitir ediciones de los mantenedores). Esto me permite corregir pequeños detalles de formato o conflictos rápidamente sin tener que pedirte cambios constantes.

### 3. Idioma
Aunque el código está en inglés, **prefiero que la comunicación (comentarios en PRs, issues) sea en español**. Si hablas español, escríbeme en español, ¡estaré encantado de atenderte así! Si no, el inglés también es bienvenido.

### 4. Seguridad y Aprobaciones
Por razones de seguridad, el repositorio está configurado para que solo el mantenedor principal pueda aprobar y mergear cambios. Cada línea de código es revisada manualmente para garantizar que no existan riesgos para los usuarios.

Gracias por tu trabajo y esfuerzo! Daremos crédito a tu aportación y aparecerás como colaborador si tu pr es mergeada. ¡Encantado de colaborar contigo!

## English

Thank you for your interest in improving CodexBar! To keep the project organized and secure, please follow these guidelines:

### 1. AI-Generated PRs
We welcome Pull Requests (PRs) created with the help of AI, under the following conditions:
- **Mandatory Verification:** You must have tested the changes locally to ensure they work.
- **Critical Review:** You must have read and understood the generated code. We will not accept "black box" code.
- **Surgical Changes:** Avoid rewriting entire files or changing indentation/formatting unnecessarily. If you are adding a single feature, change only that part. Don't "rewrite the whole world" for a small improvement.

### 2. Maintainer Edits
When opening a PR, **please make sure to check the "Allow edits from maintainers" box**. This allows me to quickly fix small formatting details or conflicts without having to ask you for constant updates.

### 3. Language
While the code is in English, **I prefer communication (PR comments, issues) to be in Spanish**. If you speak Spanish, please write to me in Spanish—I’ll be happy to reply! Otherwise, English is also welcome.

### 4. Security and Approvals
For security reasons, the repository is configured so that only the lead maintainer can approve and merge changes. Every line of code is manually reviewed to ensure there are no risks to users.

Thank you for your work and effort! We will give credit to your contribution and you will be listed as a contributor if your PR is merged. We look forward to collaborating with you!
